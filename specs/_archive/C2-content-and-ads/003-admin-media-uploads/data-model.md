# Data Model: Admin Media Uploads

## MediaFileReference

Represents an uploaded file stored on persistent disk and referenced by content or ad records.

**Fields**
- `id`: Unique media reference identifier.
- `organization_id`: Owning organization.
- `storage_path`: Internal storage reference for the file. Not exposed directly in user-facing responses.
- `public_reference`: Backend-controlled media reference used by display/admin clients.
- `original_filename`: Original uploaded filename for admin diagnostics.
- `media_type`: `image` or `video`.
- `content_type`: Validated media MIME type.
- `file_size_bytes`: Uploaded file size.
- `created_by_user_id`: User who uploaded the file.
- `created_at`, `updated_at`: Audit timestamps.

**Relationships**
- Belongs to one organization.
- May be referenced by zero or more `TopContentItem` records.
- May be referenced by zero or more `ClientAdItem` records.

**Validation Rules**
- Images must be 25 MB or smaller.
- Videos must be 500 MB or smaller.
- Media type must match the intended content/ad type.
- Storage path must be generated by the system and must not accept user-provided path traversal input.
- A media file may be deleted from disk only when no content or ad item references it.

**State Transitions**
- Uploaded -> referenced when a content/ad item successfully saves.
- Referenced -> unreferenced when all referencing content/ad items are deleted or detached.
- Unreferenced -> deleted after successful file cleanup.

## TopContentItem Changes

Represents a main display item. Existing fields remain in force.

**Added/Changed Fields**
- `media_file_id`: Optional reference to `MediaFileReference`; present for uploaded image/video content and absent for iframe content.
- `rotation_duration_seconds`: Optional per-item override for main content rotation time.
- `rotation_animation`: Optional per-item override for main content animation.
- `animation_duration_milliseconds`: Optional per-item override for animation duration.
- `source_reference`: Continues to identify the source used by the display. For uploaded files, it resolves to a protected backend-controlled media reference. For iframe content, it remains the iframe source address.

**Validation Rules**
- Uploaded image content requires a media file with `media_type=image`.
- Uploaded video content requires a media file with `media_type=video`.
- Iframe content must not require a media file and must follow approved source validation.
- Item-level rotation duration must be positive when provided.
- Item-level animation duration must be positive when provided.
- Item-level animation must be one of `none`, `fade`, or `slide` when provided.

## ClientAdItem Changes

Represents an image ad in the bottom region. Existing fields remain in force.

**Added/Changed Fields**
- `media_file_id`: Optional reference to `MediaFileReference`; present for uploaded image ads.
- `rotation_duration_seconds`: Optional per-item override for ad rotation time.
- `rotation_animation`: Optional per-item override for ad animation.
- `animation_duration_milliseconds`: Optional per-item override for animation duration.
- `source_reference`: Resolves to a protected backend-controlled media reference for uploaded image ads.

**Validation Rules**
- Uploaded ads require a media file with `media_type=image`.
- Video ads remain out of scope.
- Active ads still require an active client.
- Item-level rotation duration must be positive when provided.
- Item-level animation duration must be positive when provided.
- Item-level animation must be one of `none`, `fade`, or `slide` when provided.

## KioskDisplayConfiguration Changes

Represents default display behavior for the kiosk.

**Added Fields**
- `default_top_rotation_animation`: Default animation for main content.
- `default_ad_rotation_animation`: Default animation for ads.
- `default_top_animation_duration_milliseconds`: Default animation duration for main content.
- `default_ad_animation_duration_milliseconds`: Default animation duration for ads.
- `inline_ad_count`: Number of image ads shown inline in the ad region.

**Validation Rules**
- Default animation values must be one of `none`, `fade`, or `slide`.
- Default animation durations must be positive.
- Inline ad count must be positive.
- If inline ad count exceeds eligible ad count, the display shows only eligible ads without broken placeholders.

## Effective Rotation Settings

Represents resolved runtime display behavior for each item.

**Resolution Rules**
- For main content, use item-level rotation duration when present; otherwise use `default_top_duration_seconds`.
- For ads, use item-level rotation duration when present; otherwise use `default_ad_duration_seconds`.
- For main content animation, use item-level animation when present; otherwise use `default_top_rotation_animation`.
- For ad animation, use item-level animation when present; otherwise use `default_ad_rotation_animation`.
- For animation duration, use item-level value when present; otherwise use the matching default animation duration.

## DisplayEvent Changes

Existing display event structure remains sufficient.

**New Event Categories**
- `media_uploaded`
- `media_upload_rejected`
- `media_storage_failed`
- `media_deleted`
- `media_access_denied`
- `media_load_failed`
- `rotation_configuration_changed`

**Validation Rules**
- Event metadata must not include raw filesystem paths, session tokens, or secrets.

## Data Ownership

- All uploaded media and metadata belong to the existing single organization.
- Content managers may own and manage main content uploads within the organization.
- Advertising managers may own and manage ad uploads within the organization.
- Administrators may manage all uploaded media metadata through content/ad workflows.
- Uploaded files remain retained while at least one content or ad record references them.
